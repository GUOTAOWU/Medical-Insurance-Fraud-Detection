このプロジェクトは、「AIを活用した医療保険請求詐欺検知管理システム」です。
Python（Flask）をベースにしたWebアプリケーションであり、機械学習モデル（XGBoostなど）を使用して、医療保険の請求データが「正常」か「不正（詐欺）」かを自動的に予測・監査する機能を備えています。
全体的な機能と流れを以下に日本語で要約します。
1. プロジェクトの目的
大量の医療保険請求データの中から、不正請求（詐欺）の疑いがある案件をAIが自動的に識別し、監査担当者の業務を支援・効率化することです。
2. 主な機能構成
A. データ処理とAIモデル (Data Science & AI)
データ前処理: Excelデータの読み込み、欠損値の処理、不要な列の削除、数値の正規化（Min-Max Scaling）、カテゴリカルデータのエンコーディングを行います。
不均衡データの処理: 詐欺データは通常少ないため、SMOTE+Tomek法を用いてデータのバランスを調整し、モデルの精度を向上させています。
機械学習モデル: XGBoostやランダムフォレストなどのアルゴリズムを用いてモデルを訓練し、請求が詐欺かどうかを分類（0:正常, 1:詐欺）します。
特徴量分析: どのデータ項目（例：金額、医療機関、診断コードなど）が詐欺判定に強く影響したかを分析・可視化します。
B. Webアプリケーション機能 (Web Application - Flask)
ユーザーインターフェース（画面）を通じて、以下の機能を提供します。
1.	ユーザー認証 (Security):
ログイン/ログアウト機能。
CAPTCHA（画像認証）によるセキュリティ対策。
パスワード管理（ハッシュ化して保存）。
2.	データ管理 (Data Management):
CRUD操作: 請求データの新規登録、編集、削除、検索。
一括処理: Excelファイルによるデータの一括インポートおよびエクスポート。
ページネーション機能付きのデータ一覧表示。
3.	監査・予測実行 (Audit & Prediction):
予測実行: 新しいExcelファイルをアップロードするか、システム内の既存データを選択してAI診断を実行します。
結果表示: 「詐欺」または「正常」の判定結果をカード形式でわかりやすく表示します。
統計分析: 詐欺グループと正常グループのデータの統計的な差異（平均値や標準偏差）を表示します。
ステータス更新: AIの判定結果に基づき、担当者がその請求のステータス（承認、拒絶など）を手動で更新できます。
3. 技術スタック
バックエンド: Python, Flask, SQLAlchemy (データベース操作)
フロントエンド: HTML, Bootstrap 5 (デザイン)
データ分析・AI: Pandas, Scikit-learn, XGBoost, Imbalanced-learn
データベース: SQLite (開発用)
4. 業務フローのイメージ
1.	ログイン: 管理者がシステムにログインします。
2.	データ入力: 新しい保険請求データをExcelでインポートします。
3.	AI監査: 「監査」ページでデータを選択し、予測を実行します。
4.	結果確認: AIが「詐欺疑い」と判定した案件を確認し、詳細データや特徴量の重要度をチェックします。
5.	対応: 実際に不正かどうかを判断し、システム上でステータスを更新します。
つまり、「データの取り込みからAIによる分析、そして最終的な審査管理までをワンストップで行える業務システム」です。
